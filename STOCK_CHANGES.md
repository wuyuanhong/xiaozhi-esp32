## 一、项目概况

- **硬件**：Waveshare ESP32-S3-RLCD-4.2（400x300 单色反射屏）
- **框架**：ESP-IDF v5.5.4 + LVGL v9.4
- **音频**：BoxAudioCodec（喇叭）
- **MCU**：ESP32-S3（8MB PSRAM）

## 三、已知问题和限制

### LVGL 控件兼容性（重要！）
- **lv_label**：正常工作 ✓
- **lv_line**：50 个点以上 heap corruption 崩溃 ✗
- **lv_chart**：RLCD 上完全不工作 ✗
- **lv_canvas**：PSRAM 缓冲区渲染异常 ✗
- **结论**：只能用 lv_label，不能用图形控件

### 内存限制
- LVGL 对象过多会导致 heap corruption
- 建议股票页对象总数不超过 15 个
- `font_puhui_16_4` 是验证可用的中文字体（`alibaba_puhui_16` 字符集不全）

## 七、API 数据格式

### 腾讯财经 API
```
请求：http://qt.gtimg.cn/q=sh000001,sz399001,sh600519
返回：v_sh000001="1~上证指数~000001~3350.20~3322.66~...";
```

**字段说明（逗号分隔）**：
- [1] 名称
- [3] 当前价
- [4] 昨收
- [5] 今开
- [31] 涨跌额
- [32] 涨跌幅
- [33] 最高
- [34] 最低
- [37] 成交额（万元）

### 5. 数据获取逻辑修复

**Bug 修复**：
1. **GBK 编码问题**：腾讯财经 API 返回的中文名是 GBK 编码，`strchr(p, '~')` 会在中文字符内误匹配。重写解析逻辑，先定位纯 ASCII 的股票代码 `~XXXXXX~`，再从代码后计数 tilde。
2. **HTTP Content-Length 为 0**：API 不返回 Content-Length 头，分配固定 32KB 缓冲区读取。
3. **字段偏移量差 1**：`~000001~` 后面已经是 field[3]，需要跳 `field_offset - 3` 个 tilde。
4. **切换股票时右侧不更新**：`SwitchToNextStock` 只更新高亮，没有刷新右侧详情。修复为切换时调用 `UpdateStockLabels` 用存储的数据刷新。
5. **非选中项不可见**：列表卡片白底，非选中项白字=看不见。修复为非选中项黑字。

**新功能**：
- 切换到股票页时立即获取一次数据（不等 5 分钟间隔）
- `StockInfo` 结构体增加 `price`、`change_pct`、`open`、`pre_close`、`high`、`low`、`amount`、`turnover` 字段
- 右侧详情新增振幅和换手率显示
